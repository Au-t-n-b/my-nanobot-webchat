SDUI v3.0 协议草案（Milestone-1）

优先闭环：①局部增量更新（Patch）④图表交互（下钻联动预览）⑤韧性（Skeleton/局部降级）。目标：工勘大盘高频同步“无闪烁”，点击图表→自动打开右栏预览。

v2 vs v3：能力对比（摘要）
对比表
主题 v2（现状） v3（Milestone-1）
更新粒度 全量下发 SduiDocument Patch：按 nodeId / path 做局部替换/合并
交互 SduiAction 仅 open_preview/post_user_message 事件绑定：图表 segment/datum 绑定 action，点击触发 open_preview 或回传

1) SduiPatch：JSON 结构（草案）

原则：后端不做复杂 diff，只发送“稳定 id 定位 + 局部节点/属性”。前端负责按规则 merge，确保组件实例不销毁。

建议新增：SduiEnvelope（统一承载 full/patch/batch）
{
  "schemaVersion": 3,
  "type": "SduiEnvelope",
  "docId": "dashboard:gc",              // 可选：同一大盘稳定 ID，用于 patch 归属/回放
  "revision": 128,                     // 单调递增（或时间戳），用于乱序丢弃
  "payload": {
    "kind": "document",                // 或 "patch" / "batch"
    "document": { ...SduiDocument... } // v2 文档原样复用（root/meta/node.type 不变）
  }
}
SduiPatch（path-based / id-based 二选一或并存）
{
  "schemaVersion": 3,
  "type": "SduiPatch",
  "docId": "dashboard:gc",
  "baseRevision": 127,                 // 可选：乐观并发；不匹配则前端可请求全量或丢弃
  "revision": 128,
  "ops": [
    {
      "op": "merge",                   // merge | replace | remove
      "target": { "by": "id", "nodeId": "stat-1" },
      "value": { "type": "Statistic", "id": "stat-1", "value": "95%" },
      "anim": { "kind": "numberTick" } // 可选：提示宿主做数字跳变
    },
    {
      "op": "merge",
      "target": { "by": "path", "path": "/root/children/2/tabs/0/children/1" },
      "value": { "type": "Text", "content": "更新于 12:01" }
    }
  ]
}

建议约束（确保“无闪烁”）：Patch 默认只能更新“叶子节点的展示字段”（如 Statistic.value/Text.content/Badge.label/Chart.data），不允许改动结构性 children（Milestone-1）。结构变更仍走全量 document。

3) 向后兼容：v2 文档与 v3 Patch 共存策略

兼容规则（建议）

  • v2 文档不变：SduiDocument(schemaVersion=1) 继续可渲染
  • v3 引入 Envelope：宿主优先解析 Envelope；否则按旧逻辑 parse doc
  • Patch 仅作用于指定 docId：没有对应 docId 的基线文档则丢弃/请求全量
  • Revision 单调递增：乱序/重复 patch 直接丢弃

版本演进（里程碑）

  • v3.0（M1）：叶子字段 patch + 图表 open_preview + skeleton
  • v3.1：emit_event / client-state / expressions（你提的 #2）
  • v3.2：动态表单（#3）
