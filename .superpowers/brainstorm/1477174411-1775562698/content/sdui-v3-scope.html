SDUI v3.0：从“展示协议”到“交互协议”

范围：同时覆盖右栏预览体系（文件 / browser / skill-ui / tabs / 阻断层）与 SDUI。请先选一个“最先闭环”的方向。

你提出的 5 个 v3 方向
1

局部差量更新（Patch/Incremental Update）

减少全量下发、避免闪烁；适合大盘高频刷新。

2

客户端简单逻辑（Local State + Expressions）

纯 UI 交互零 RTT；但需要最小表达式语言与安全沙箱。

3

动态表单（Form Schema）

级联选择、校验、分组输入；从“对话输入”升级为“工单填写”。

4

图表交互（事件绑定 / 下钻）

点击扇区/柱子触发 open_preview / state 更新 / 查询回传。

5

韧性（骨架屏 / 错误边界 / 局部降级）

加载阶段更“有质感”，解析失败不拖垮整棵树。

交互闭环（B 范围）我们要对齐的关键点

必须覆盖

  • 右栏 Tabs：dashboard / blocking / previews
  • open_preview：文件 / browser / skill-ui 统一入口
  • SDUI：输入 → 提交 → 局部更新/预览联动

需要谨慎

  • 表达式执行与可控能力（不能变成脚本注入）
  • Patch 语义与一致性（顺序、冲突、回放）
  • 状态域划分（document state / preview state / global state）
