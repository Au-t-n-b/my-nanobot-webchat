# 智慧工勘模块案例 迁移说明

## 模块目标
把 zhgk 的智慧工勘旧版业务流提炼为一个模块案例：先开大盘，再通过勘测场景选择与资料上传推进流程，并让右侧进展、黄金指标、产物总结实时联动。

## 本次模块案例输入
- 勘测场景：原址扩容
- 资料包：深圳A03站点_BOQ与勘测包.zip

## 提取后的业务主线
1. 场景过滤：根据 BOQ / 预置集识别制冷方式、确认勘测场景，并初始化定制化底表。
2. 勘测汇总：汇总勘测结果、现场图片和补充材料，输出全量勘测结果表与待办清单。
3. 报告生成：生成满足度评估表、风险识别结果表和工勘报告。
4. 审批分发：把最终报告发送给专家审批，再通知干系人闭环。

## 样板大盘指标
- 资料完整度：92
- 报告就绪度：86
- 风险关注项：4

## 给业务同事的迁移建议
1. 继续保留 guide/start/choose_strategy/upload_evidence/after_upload/finish 的模块节奏。
2. 把本模块里的“勘测场景选择”替换为你们业务自己的关键 HITL 决策点。
3. 上传卡片建议统一收敛到资料包、底表或外部系统导出文件，避免回退为纯文本输入。
4. 所有阶段都要通过 SkillUiDataPatch 更新 Stepper、黄金指标和产物区，确保大盘实时同步。
